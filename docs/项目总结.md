# 基于产生式知识表示的专家系统

## 项目概述

基于产生式规则的专家系统，采用纯 Python 推理引擎 + PyQt6 GUI + Vue3 Web 前端架构。

## 技术架构

| 模块     | 技术                | 文件                   |
| -------- | ------------------- | ---------------------- |
| 推理引擎 | Python              | `src/core/reasoner.py` |
| 数据存储 | JSON                | `src/data/storage.py`  |
| 桌面 GUI | PyQt6               | `src/gui/`             |
| Web 后端 | Flask               | `src/web/server.py`    |
| Web 前端 | Vue3 + Element Plus | `web_frontend/`        |

## 主要功能

- 正向推理：从已知事实推导结论
- 反向推理：从目标结论反向验证
- 推理可视化：vis-network 绘制推理路径图
- 知识库管理：规则增删改查
- 用户系统：注册登录、权限控制
- 历史记录：保存推理结果

## 部署方式

```bash
# 安装依赖
uv sync

# 构建前端
cd web_frontend && pnpm install && pnpm build && cd ..

# 启动 GUI
uv run python main.py

# 仅启动 Web 服务器
uv run python main.py --web --port 5000
```

## 项目结构

```
├── main.py                 # 主入口
├── src/
│   ├── core/reasoner.py    # 推理引擎
│   ├── data/storage.py     # 数据存储
│   ├── gui/                # GUI 组件
│   └── web/server.py       # Web API
├── web_frontend/           # Vue3 前端
└── pyproject.toml          # 项目配置
```

## 默认规则库

动物识别专家系统示例：毛发 → 哺乳动物、羽毛 → 鸟、哺乳动物+肉食动物+黄褐色+黑色条纹 → 老虎 等。

## 技术亮点

- 纯 Python 实现，无需编译，跨平台运行
- 模块化架构，职责分离清晰
- 双端支持：桌面 GUI 和 Web 界面
- uv 管理依赖，简化部署流程
